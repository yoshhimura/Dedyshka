1. Общее описание

Система реализуется как монолитное веб-приложение на основе архитектуры "клиент-сервер". Все компоненты (frontend, backend, database) работают в рамках одного приложения и развертываются вместе. Это упрощает разработку, тестирование и деплой на начальных этапах проекта.

Приложение состоит из трёх основных слоёв:

Frontend — отвечает за пользовательский интерфейс.
Backend — обрабатывает логику приложения и взаимодействует с базой данных.
Database — хранит все данные приложения.

3. Основные API endpoints
Backend предоставляет следующие RESTful API endpoints:

Управление задачами
GET /api/tasks — Получить список всех задач (с фильтрацией и поиском)
GET /api/tasks/<int:task_id> — Получить одну задачу по ID
POST /api/tasks — Создать новую задачу
PUT /api/tasks/<int:task_id> — Обновить существующую задачу
DELETE /api/tasks/<int:task_id> — Удалить задачу
Управление категориями
GET /api/categories — Получить список всех категорий
POST /api/categories — Создать новую категорию
PUT /api/categories/<int:category_id> — Обновить категорию
DELETE /api/categories/<int:category_id> — Удалить категорию
Фильтрация и поиск
GET /api/tasks?priority=high&category=work — Фильтрация по приоритету и категории
GET /api/tasks?search=отчет — Поиск задач по названию или описанию
Дедлайны и напоминания (упрощённо)
PUT /api/tasks/<int:task_id>/reminder — Включить/выключить напоминание для задачи
GET /api/tasks?due_today=true — Получить задачи, дедлайн которых сегодня
